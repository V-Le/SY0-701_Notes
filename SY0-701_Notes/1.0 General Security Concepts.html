1.0 General Security Concepts

1.1 Compare and contrast various types of security controls
...

Control Categories
...

Technical Controls

  • Technological measures embedded within the system, hardware, or software.
  • Uses technology and automated mechanisms to enforce security.
  • Directly enforces security policies by controlling access, protecting data, and preventing or detecting security threats.
  • i.e., firewalls, antivirus software, intrusion detection systems, etc.

Managerial Controls

  • Policies, procedures, and guidelines established by management to guide the organization’s security practices. Security Design and implementation.
  • Concerned with management, planning, and oversight aspect of security including policies, risk assessments, training, and compliance.
  • Provides a framework for governance and guides the organization’s approach to security and ensures alignment with business objectives.
  • i.e., security policies, security awareness training, incident response plans, etc.

Operational Controls

  • Security implemented by people instead of systems.
  • Involves the implementation and management of security processes, monitoring, and ongoing maintenance.
  • Ensures that security processes are properly executed, incidents are handles effectively, and ongoing security measures are maintained.
  • i.e., security guards, lunch and learns, posters spreading awareness.

Physical Controls

  • Measures implemented to protect physical assets, facilities, and resources from unauthorized access or damage.
  • Involves tangible security measures that safeguard physical assets, including buildings, equipment, servers, etc.
  • Protects physical assets, premises, and resources from unauthorized access, theft, damage, or disruption.
  • i.e., fences, lockers, gate, etc.

Control Types
...

Preventative - Blocks or limits access to a resource.

  • Technical: Firewall rules
  • Managerial: On-boarding policy
  • Operational: Guard shack personnels checking all identification
  • Physical: Active door lock

Deterrent - Discourages an intrusion attempt but does not directly prevent access.

  • Technical: Application splash screen
  • Managerial: Threat of demotion
  • Operational: Reception desk
  • Physical: Posted warning Signs

Detective - Identify and log an intrusion attempt but does not prevent access.

  • Technical: Collect and review system logs
  • Managerial: Person review login reports
  • Operational: Regularly patrolling the property
  • Physical: Active motion detectors

Corrective - Applies a control after an event that has been detected to continue operation with minimal downtown.

  • Technical: Restoring from backup can mitigate an attack
  • Managerial: Create policies for reporting issues
  • Operational: Contacting authorities
  • Physical: Fire extinguisher for fires

Compensating - Temporary control using other means because existing controls aren’t sufficient.

  • Technical: Blocking a specific application instead of patching
  • Managerial: Implementing a separation of duties
  • Operational: Require multiple security staff
  • Physical: Power generator after a power outage

Directive - Directing personnel towards security compliance or “Please do this”. A relatively weak security control.

  • Technical: Store sensitive files in a protected folder
  • Managerial: Create compliance policies and procedures
  • Operational: Train personnel on proper security policies
  • Physical: Post a sign for “Authorized Personnel Only”

1.2 Summarize fundamental security concepts
...

Confidentiality, Integrity, and Availability (CIA)
...

The fundamentals of Security, sometimes referenced as AIC Triad.

Confidentiality - Ensuring that information is accessible only to those who have the proper authorization, preventing unauthorized access.

  • Encryption - Encode messages so only certain people can read it.
  • Access Controls - Selectively restrict access to a resource.
  • Two-Factor Authentication - Additional confirmation before information is disclosed.

Integrity - Ensures the accuracy and reliability of data throughout its lifecycle. It ensures that data is not altered or tampered with during transmission or storage.

  • Hashing - Map of data to an arbitrary length to data of a fixed length
  • Digital Signature - Mathematical scheme to verify the integrity of data.
  • Certificates - Combine with digital signature to verify an individual.
  • Non-Repudiation - Provides proof of integrity and confirm to be genuine.

Availability - Ensures that information and resources are accessible and available to authorized users whenever needed. It involves preventing disruptions or downtime that could impact access.

  • Redundancy - Build services that will always be available.
  • Fault Tolerance - System will continue to run, even when failure occurs. If one component fails, then secondary backup component will be used.
  • Patching - Managed and updated to ensure systems are stable and prevents access due to exploits.

Non-Repudiation
...

Ensures that a user cannot deny the authenticity of their actions or the transactions they have performed.

Proof of Integrity - Verify that data does not change and remains accurate and consistent.

Hash

  • A short string of text used to represent data, AKA message digest, a fingerprint.
  • If data changes, the hash text changes too.
  • Doesn’t associate data with individual, only tells you if data has changed.

Proof of Origin

  • Integrity - Proof that the message was not changed.
  • Authentication - Prove the source where the data was sent.
  • Non-Repudiation - Verify the signature isn’t fake.
  • Digital Signature.
    • Private Key - Digital signature only known to the person sending the data.
    • Public Key - Associated with the private and verifies the information is the same data and person that provided the digital signature.

Authentication, Authorization, and Accounting (AAA)
...

AAA Framework

  • Identification - Who you claim to be.
    - Usually your username.
  • Authentication - Prove who you say you are.
    - The check between your user name, password, and other types of authentications.
  • Authorization - The type of accesses the user has on a system.
    - Based on the identification and authentication.
  • Accounting - A log of exactly what happened.
    - Resources used: Login time, data sent, data received, logout time.

Authenticating People - Verification of an individual's identity using various methods and technology. Ensures only authorized individuals can access systems, data, or physical locations.

  1. Username and Password - Individual provides a unique username and secret password as a means of authentication.
  2. Multi-Factor Authentication (MFA) - Individual provides two or more types of identification.
  3. Biometric Authentication - Individual provides unique physical or behavioral characteristics.
  4. Smart Cards and Tokens - Individual uses physical cards or tokens contain embedded information or cryptographic keys.
  5. Mobile Authentication - Utilizes mobile devices for authentication through apps, SMS codes, or push notifications.
  6. Digital Certificates - Involves use of digital certificates issued by a trusted Certificate Authority (CA) to verify identity.
  7. Knowledge-Based Authentication (KBA) - Requires individuals to answer knowledge-based questions, often related to personal information.
  8. Location-Based Authentication - Authenticates individuals based on their geographical location or the location of the device they are using.

Authenticating Systems - Mechanisms and process implemented to verify the identity of users, devices, or entities attempting to access a computer system, network, or application.

  1. Username and Password - Credentials are compared to a database before allowing access
  2. Multi-Factor Authentication (MFA) - Requires user to provide two or more types of identification.
  3. Biometric Authentication - Requires unique physical or behavioral characteristics of individuals.
    • Fingerprints, facial recognition, iris scans, voice recognition, etc.
  4. Token-Based Authentication - Requires use of physical or virtual tokens.
    • Smart cards, USB tokens, mobile apps generating time-based codes, etc.
  5. Certificate-Based Authentication - Utilizes digital certificates issued by a trusted Certificate Authority (CA) to verify authenticity of users or devices.
  6. Single Sign-On (SSO) - Enables users to log in once and access multiple systems or applications without re-entering credentials.
  7. Kerberos Authentication - A network authentication protocol that uses a third-party Key Distribution Center (KDC) to validate the identity of users and provide secure session keys.

Authorization Models - Defines the process of granting of denying access to resources based on the authenticated identity of individuals or entities.

Gap Analysis
...

A method used to assess the difference, or "gap", between a current state and a desired future state in a particular process, system, or organization.

  • Used to evaluate the security measures in place compared to industry standards, best practices, or regulatory requirements.
  • Helps organizations identify vulnerabilities, weakness, and areas where improvements are needed to enhance their security.
  • This process helps prioritize and implement measures to bridge the identified gaps and improve overall cybersecurity resilience.

Choosing the framework

  1. Work towards a known baseline
    • May be an internal set of goals
  2. Evaluate baseline of employees
    • What formal experience do they have with security?
    • What is the current training?
    • What knowledge they have of security policies and procedures?
  3. Evaluate baseline of current processes.
    • Research existing IT systems.
    • Evaluate existing security policies.
  4. Determine the end goal
    • Some organizations should use formal standards. i.e., NIST Special Publication 800-171 R.2, ISO/IEC 27001.

Compare and Contrast

  1. Comparison
    • Evaluate existing systems.
  2. Identify weaknesses and vulnerabilities
    • Along with most effective processes to how to compensate.
  3. Create Detailed analysis
    • Examine broad security categories.
    • Break into smaller segments.

Analysis and Report

  1. Final Comparison
    • Detailed baseline objectives.
    • A Clear view of current state.
  2. Determine best path to get current security state to end goal
    • Will take time, money, and lots of change control.
  3. Create Gap analysis report
    • A formal description of the current state.
    • Recommendations for meeting the end goal baseline starting with highest priority risks.

Zero Trust
...

Many networks are relative open on the inside. Once you're through the firewall, there are few security controls. Zero Trust is is a holistic approve to network security covering every device, process, and person where everything must be verified even once past the firewall or initial authentication.

Planes of Operation
Splits the network into functional planes. Applies to physical, virtual, and cloud components.

Control Plane - Defines the policies and rules, and manages the actions of the data plane. (i.e., the brain telling the hand to grab and object and move it)

  • Adaptive identity - Considers the source and requested resources using multiple risk indicators(i.e., relationship to organization, physical location, type of connection, IP address, etc.), and automatically makes authentication stronger if needed.
  • Threat scope reduction - Decrease number of possible entry points.
  • Policy-driven access control - An approach to access management where access to resources is governed by predefined policies. These policies, centrally defined and often granular(high level of detail), dictate the conditions under which access permissions are granted or denied.
  • Policy Engine - A software component or system that evaluates each access decision based on a predefined policy and makes a decision if the access is granted, denied, or revoked.
  • Policy Administrator - Provides the decision of the Policy Engine to the Policy Enforcement Point(PEP). May also generate access tokens or credentials to provide to the Policy Enforcement Point(PEP).

Data Plane - Actively enforces the security policies set by the control plane. Does the actual processing of frames, packets, or network data. (i.e., the hand actually grabbing the object and moving it)

  • Security Zones - Refers to a logical or physical segment of a network that is subject to a specific set of security policies and controls. It is used to compartmentalize and organize network resources based on their security requirements and the level of trust assigned to entities within each zone. Looks at where you are connecting from and connecting to(i.e., a personal computer over the internet(untrusted zone) will be denied access to an company network(trusted zone) until AAA requirements are met).
    • Implicit Trust Zones - A network zone where all communications are assumed to be secure and free of malicious actors or activities(i.e., an authorized device on a company network(implicitly trusted) freely accessing a company resource within the company network(trusted zone)).
  • Subject/System - Refers to end users, applications, non-human entities, etc.
  • Policy Enforcement Point (PEP) - A component within a network or information security system that is responsible for enforcing access control policies and makes decisions based on predefined policies. The PEP acts as a gatekeeper, monitoring access requests and allowing or denying access based on the established policies.
  • Policy Decision Point (PDP) - Often works in conjunction with the PDP. The PDP is responsible for defining and managing access control policies, while the Policy Decision Point enforces the policies.

Physical Security
...

Bollards & Barricades - Prevents access with certain limitations depending on type.

  • Channel people through specific access points.
  • Can be used as a security or safety notice by using colors.
  • i.e., Bollards allows people, prevents vehicles and trucks. Moat barricade funnel people through a bridge.

Access Control Vestibule - A security room where an individual must pass through to gain access to the rest of the facility. Also known as mantraps or security vestibules. Typically located at entry points of buildings or secure areas.

  • Typically equipped with a form of controlled access points. (i.e., turnstiles, card access systems, two-door interlock system, front desk check-in/check-out)
  • Manages authentication through an area.

Fencing - Builds a perimeter around a restricted area that prevents access.

  • Usually obvious and easily seen.
  • Can be seen through or not (chicken wire fence, house wooden fence)
  • Should be robust and not easily broken.
  • Can include razor wire, spikes, built high, etc. to prevent individuals from climbing over.

Video Surveillance - Closed circuit television (CCTV)

  • Can replace physical guards.
  • Can have features such as motion detection, detailed object detection that can identify license plate or person's face.
  • Often many different cameras networked together and sends data back to a single storage point.

Security Guard - A person that provides physical protection at an access point.

  • Validates identification of existing employees
  • A two-person integrity or control minimizes attacks so that not one person has full access

Access Badge - A badge that identifies an individual.

  • Usually includes a picture, name, and/or other details.
  • Must be worn at all times.
  • Often integrated with electronic locks that can log access to a central database.

Lighting - Illuminating an area allows for clearer view of an area.

  • Easier to see when lit.
  • Attackers avoid the light.
  • Consider overall lighting and angles for the area.

Sensors -

  • Infrared - Detects infrared radiation in both light and dark. Don't need light
  • Pressure - Detects changes in force
  • Microwave - Detects movement over a large area
  • Ultrasonic - Sends ultrasonic signals and receive reflected sound waves. Detects motion.

Deception and Disruption Technology
...

Honeypot - A mechanism designed to detect, deflect, or study attempts at unauthorized use of information systems.

  • Consists of a system or network that appears to be a legitimate environment but is actually isolated, monitored, and intentionally left vulnerable.
  • The primary purpose is to attract and deceive attackers, allowing security professionals to observe the attacker's tactics, techniques, and motives.

Honeynet - A network of combined honeypots designed to work together to detect and analyze malicious activity on a larger scale. Consists of multiple honeypots deployed within a isolated environment.

  • Collective data and insights gained from honeynets provide a boarder and more comprehensive understanding of cybersecurity threats.
  • Valuable tools for research, threat intelligence, and detection of more sophisticated attacks.

Honeyfile - Decoy files or data intentionally placed within a system or network to act as bait.

  • Files are designed to mimic legitimate data or resources that entice attackers to interact with them, revealing their presence and tactics. i.e., passwords.txt
  • Particularly useful for identifying unauthorized access, lateral movement within a network, or attempts to gather information.
  • Can send an alert if file is accessed.

Honeytoken - Decoy unique data intentionally placed within a system or network to act as bait.

  • Adds some traceable data to the honeynet. Stolen data can be traced by to source of breach.
  • i.e., fake API credentials that send back notifications when used, fake email addresses added to a spam list, database records, browser cookies, webpages, etc.

1.3 Explain the importance of change management processes and the impact to security
...

Business Processes Impacting Security Operation
...

Approval Process - Maintains uptime and availability of systems. Avoids downtime, confusion, and mistakes.

  • A typical approval process:
    1. Complete request form
    2. Determine purpose of change
    3. Identify scope of change
    4. Schedule date and time
    5. Determine affected systems and impact
    6. Analyze risk associated
    7. Get approval from change control board
    8. Test change and get end-user acceptance and feedback

Ownership - Individual or department that decides a change is needed.

  • Owns and manages the process.
  • Is updated on the process to ensure processes is followed and acceptable.
  • Doesn't control the change control process.
  • Doesn't usually perform the actual change.
  • IT handles the actual change.
  • i.e., department decides it needs to upgrade shipping label software -> department creates a request form -> department hands it off to an IT team to handle actual change.

Stakeholders - Individuals or departments that will be impacted by the change.

  • May want to have input on the change management process.
  • Single changes may include one individual or entire company.
  • i.e., upgrades to shipping labels -> affects accounting to create reports -> may affect product delivery timeframes -> affects revenue due to shipping delivery speeds -> affected revenue would have CEO visibility.

Impact Analysis - Involves evaluating the potential effects and consequences of a proposed change in an organization's system or processes.

  • Determine a risk value. (i.e., High, Medium, low, or Severe, Moderate, Mild, etc.)
  • Risk can be minor or far-reaching. (i.e., "fix" may not actually fix anything., fix may break something, operation system failure, data corruption)
  • Consider the risk of not making the change. (i.e., security vulnerability, application may be unavailable, etc.)

Test Results - Ensure change works properly before pushing to production.

  • Create a sandbox testing environment.
    • Has no connection to the real world or production system.
    • A technological safe space.
    • Can recreate same environment as production system.
  • Test changes in a sandbox before making change to production.
    • Try the upgrade, apply the patch, test any changes that need to be made to production.
    • Test and confirm before actual deployment on production system.
  • Test and confirm backout/rollback plan.
    • Document steps to how to rollback changes.
    • Sandbox does not account for every possibility.

Backout Plan - A plan that reverts back to the original production system.

  • Changes don't always go as planned, so always have a way to revert changes.
  • Sometimes as easy as uninstall the application, sometimes much more difficult.
  • A backout plan minimizes downtime in the event the change breaks production.
  • Backup often in case a backup of the backup is needed.

Maintenance Window - A predefined period of time during which planned changes, updates, or maintenance activities are scheduled to be implemented.

  • Typically chosen to minimize disruption to normal business operations.
  • Potential downtime could affect a large part of production.

Standard Operating Procedure (SOPs) - Documented guidelines that outline the step-by-step processes and protocols to be followed when planning, implementing, and evaluating changes within an organization's systems or processes.

  • Following standardized procedures ensures consistency, transparency, and security.
  • i.e., a SOP could include detailed procedures for submitting change requests, assessing impacts through impact analysis, obtaining necessary approvals, scheduling maintenance windows, implementing changes, and conducting post-implementation reviews.

Technical Change Management and Implications
...

A subset of change management that specifically deals with changes in a technical aspect of an organization, such as IT infrastructure, systems, applications, and cybersecurity measures. It is more narrowly focused on ensuring that technical changes are handled effectively, securely, and with minimal disruption.

Allow Lists/Deny Lists - Access control mechanisms used to define which entities (users, applications, devices) are permitted or denied access to certain resources or actions. Commonly employed to enhance security by controlling and restricting access based on predefined criteria.

  • Allow List (Whitelist) - List of entities that are explicitly permitted or allowed access to a particular resource or system. Only entities on the Allow List are granted access, and all others are denied by default. (i.e., a firewall Allow List that specifies which IP address are allowed to connect to a network)
  • Deny List (Blacklist) - List of entities that are explicitly prohibited or denied access to a specific resource or system. Any entity on the Deny List is not allowed execution or access, and all others are allowed by default. (i.e., an anti-virus Deny List that blocks access or execution of known malicious files or websites)

Restricted Activities - Actions or processes that are subject to additional controls, scrutiny, or limitations due to potential impact on the organizations technical infrastructure, systems, or security.

  • These activities are typically considered high-risk, and careful management is necessary to avoid adverse consequences.
  • Change Control approval is very specific to the scope of change and defines exactly which components are covered.
  • The scope may need to be expanded during the change window.

Downtime - The period during which systems, services, or applications are not available or operational. Occurs when planned changes, upgrades, maintenance, or other technical activities are being carried out and affected systems or services are temporarily taken offline or unavailable to users.

  • Usually scheduled during non-production hours to minimize amount of downtime.
  • Mitigation of downtime can consist of temporarily switching to a secondary system.
  • Early notice to users is given in the event of planned downtime (email, maintenance calendar, update roadmap).

Restarts - Restarts or reboots are common to implement new configurations. Restarts include rebooting the OS, power cycling a physical device, or restarting a service.

  • Service Restart - Stop and restart the service or daemon.
  • Application Restart - Closing the application completely and relaunch a new application instance.

Legacy Applications - Software applications that have been in use for a long period and are often considered outdated or obsolete. These applications may still be functional and serve a purpose within an organization, but they are typically built on older technologies, frameworks, or programming languages.

  • Often no longer supported by the developer.
  • Creating specific processes and procedures and documenting the application and how it is installed can help with supporting the application.

Dependencies - The relationship and connections between difference components, tasks, or modules. These dependencies indicate the order or sequence in which activities need to be completed or the reliance of one component on another.

  • Finish-to-Start (FS) - One task must finish before another can start, i.e., Task B depends on the completion of Task A
  • Start-to-Start (SS) - Two tasks must start simultaneously. i.e., Task A is dependent on the start of Task B

Documentation
...

The process of capturing, storing, and communicating information about a system, process, product, or project. This involves creating records, manuals, guides, or any other written or visual materials that provide details, instructions, specifications, and context for understanding and using a particular entity.

Updating diagrams - Updating documentations and diagrams to network configurations or charts.

Updating policies/procedures - Updating documentation for policies and procedures during any changes, updates, or upgrades.

Version Control
...

Also known as source control or revision control, is a system that manages changes to documents, source code, or any set of files over time. It allows multiple contributors to work collaboratively on a project, track changes, and maintain a history of revisions.

  • Facilitates tracking of changes and the ability7 to view the entire history of changes made to files, including who made the changes and when.
  • Provides a mechanism for reverting to previous states if needed.
  • May require additional management software.

Public Key Infrastructure (PKI)
...

A comprehensive system that manages the creation, distribution, storage, managing, and revocation of digital keys and certificates. It is a set of roles, policies, hardware, software, and standards that work together to provide a framework for secure communication and authentication over a network, typically the internet.

  • Widely used in securing communication over the internet, including secure email, web browsing, and Virtual Private Networks (VPN).
  • Plays a crucial role in establishing trust and ensuring the confidentiality, integrity, and authenticity of digital transactions.

Symmetric Encryption - A cryptographic method where the same key is used for both the encryption and decryption of data.

  • The sender and the recipient share a single secret key that is used to both encrypt and decrypt the information.
  • Also known as a Shared Secret.

Asymmetric Encryption - Also known as Public-Key Cryptography, is a cryptographic method that uses a pair of keys for secure communication: a public key and a private key. These keys are mathematically related but serve a different purpose.

  • Public Key - A cryptographic key that is publicly shared and used for encryption or verifying digital signatures.
  • Private Key - A cryptographic key that is kept secret and used for decrypting information or creating digital signatures.

Key Escrow - The practice of storing a copy of a user's private encryption key with a trusted third party, known as the key escrow agent.

  • This provides a mechanism for recovery in case the user loses access to their private key or in situations where access to encrypted data is required for legal or regulatory reasons.

Encryption
...

Level - The different scopes or units where encryption can be applied to protect data.
Full-Disk, Partition, and Volume encryption can be use to protect data on storage devices. Also known as encrypting data at rest. i.e., BitLocker, FileVault, etc.

  • Full-Disk Encryption - This involves encrypting the entire storage disk or drive. All data on the disk is automatically encrypted and requires decryption when the system is started.
    • Use Case: Protecting all data on a computer or device in case of loss or theft.
  • Partition Encryption - Similar to full-disk encryption but focuses on encrypting specific partitions on a storage device rather than the entire disk.
    • Use Case: Providing selective encryption for specific partitions containing sensitive data.
  • Volume Encryption - Similar to partition encryption. volume encryption involves encrypting specific logical volumes. This could be applied to parts of a storage system that are treated as separate volumes.
    • Use Case: Protecting distinct volumes with specific data.
  • File Encryption - Encrypting individual files or directories. Each file is encrypted separately.
    • Use Case: Securing specific files or folders containing sensitive information.
  • Database Encryption - Encrypting the entire database or specific tables within a database to protect the stored data.
    • Use Case: Enhancing the security of databases containing sensitive information.
  • Record Encryption - Encrypting individual records or rows within a database. Each record is encrypted independently.
    • Use Case: Providing granular security for specific pieces of information within a database.

Transport/Communication Encryption - Encryption applied to the communication between two entities, typically over a network, to secure the data transmitted between them.

  • Use Case: Protecting data during transmission over the internet, ensuring confidentiality and integrity. i.e., browsers using HTTPS, using a Virtual Private Network(VPN)

Asymmetric Encryption - Encryption method that uses a pair of keys - a public key for encryption and a private key for decryption. They keys are mathematically related but cannot be derived from each other.

  • Use Case: Securely exchanging keys. creating digital signatures, and establishing secure communication channels

Symmetric Encryption - Encryption method where the same key is used for both encryption and decryption. It is computationally efficient for large volumes of data.

  • Use Case: Securing data at rest, where the key must be kept secret between the communication parties.

Key Exchange - The process of securely transmitting cryptographic keys between parties to enable secure communication. Key exchange mechanisms ensure that only authorized parties possess the necessary keys for encryption and encryption.

  • Use Case: Establishing a shared secret key between parties before initiating encrypted communication.

Algorithms - Mathematical procedures or rules used in encryption to transform plaintext into ciphertext or vice versa. Encryption algorithms can be symmetric or asymmetric.

  • Use Case: Determining the specific cryptographic operations performed during encryption and decryption.

Key Length - The size of the cryptographic key used in encryption algorithms, measured in bits. Longer key lengths generally provide stronger security but may require more computational resources.

  • Use Case: Choosing an appropriate key length based on security requirements and considering the trade-off between security and performance.
